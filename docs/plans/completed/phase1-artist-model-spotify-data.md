# Plan: Phase 1 — Artist Model + Expanded Spotify Data

**Status:** Done
**Created:** 2026-03-22

## Goal

Normalize artists into their own table, expand the Spotify data we extract (popularity, duration, release date), and update the sync pipeline to handle the new schema with chunked processing. This is the foundation for the enrichment pipeline — all later phases depend on the Artist model and enrichment version columns.

Since this is a solo project with only dev data, we use a destructive migration (truncate `track` + `liked_song`) to avoid backfill complexity. Each PR keeps main green.

- [x] PR 1: Artist schema + Spotify extraction + repository updates — `feat/artist-model-spotify-data`
- [x] PR 2: Chunked sync pipeline + artist repository — `feat/chunked-sync-pipeline`

**Schema changes:**
- New `Artist` model: `id` (cuid), `spotifyId` (@unique), `name`, `spotifyGenres` (String[]), `lastfmTags` (String[]), `enrichmentVersion` (Int, default 0), `enrichedAt` (DateTime?)
- New `TrackArtist` join model: `trackId`, `artistId`, `position` (Int). Composite PK on `(trackId, artistId)`.
- Track: drop `artist` (String), `bpm` (Int?), `era` (String?)
- Track: add `spotifyPopularity` (Int?), `spotifyDurationMs` (Int?), `spotifyReleaseDate` (String?), `derivedEra` (String?), `enrichmentVersion` (Int, default 0), `enrichedAt` (DateTime?)
- Track: change `lastfmGenres` (String?) → `lastfmTags` (String[])

**Migration strategy:**
- Migration SQL: DROP TABLE `liked_song`, DROP TABLE `track` (cascade), then CREATE fresh `track`, `artist`, `track_artist`, `liked_song` tables.

**Domain type changes (`src/domain/song.ts`):**
- `Track`: remove `artist`, `bpm`, `lastfmGenres`, `era`. Add `spotifyPopularity`, `spotifyDurationMs`, `spotifyReleaseDate`, `derivedEra`, `lastfmTags`, `enrichmentVersion`, `enrichedAt`.
- `TrackWithLikedAt`: extend Track & add `artist: string` + `likedAt: Date`. The `artist` field is populated via STRING_AGG in the query layer, not from a column.
- New `Artist` type: `id`, `spotifyId`, `name`, `spotifyGenres`, `lastfmTags`, `enrichmentVersion`, `enrichedAt`, `createdAt`, `updatedAt`
- New `TrackArtist` type: `trackId`, `artistId`, `position`

**Spotify extraction changes (`src/lib/spotify.ts`):**
- `SpotifyLikedTrackItem`: add `track.popularity`, `track.duration_ms`, `track.album.release_date`, `track.artists[].id`
- `SpotifyLikedSong`: drop `artist: string`. Add `spotifyPopularity`, `spotifyDurationMs`, `spotifyReleaseDate`, `artists: { spotifyId: string; name: string }[]`
- `mapTrack()`: return structured artists array + new fields
- `fetchLikedSongs`: signature stays unchanged for now (still returns flat `SpotifyLikedSong[]`, exhausts all pages). Chunking comes in PR 2.

**Track repository changes:**
- `upsertMany(userId, songs)` — expanded to handle artists in each 500-song batch:
  1. Deduplicate artists from songs → bulk INSERT artist ON CONFLICT UPDATE name
  2. SELECT artist IDs by spotifyId → build map
  3. Bulk INSERT track ON CONFLICT UPDATE (name, album, albumArtUrl, spotifyPopularity, spotifyDurationMs, spotifyReleaseDate)
  4. SELECT track IDs by spotifyId → build map
  5. Bulk INSERT track_artist ON CONFLICT DO NOTHING
  6. Bulk INSERT liked_song ON CONFLICT DO NOTHING
- `findByUserId` — join through `trackArtist` → `artist`, use raw SQL or Kysely expression for STRING_AGG ordered by `trackArtist.position`. Return type stays `TrackWithLikedAt[]` — frontend sees no change.

**Sync pipeline (`sync-library.ts`):**
- No changes needed. It calls `fetchLikedSongs` (same signature) and `trackRepository.upsertMany` (same signature, richer internals). The `upsert-songs` step name stays for now.

**Test updates:**
- `tests/lib/spotify.test.ts`: update `makeSpotifyItem` helper to include artist ID, popularity, duration, release date. Update assertions for structured artists array.
- `tests/repositories/track.repository.test.ts`: update mock song data passed to `upsertMany` (structured artists instead of string). Verify `insertInto` called with `artist` and `trackArtist` tables.
- `tests/inngest/functions/sync-library.test.ts`: update mock `fetchLikedSongs` return data (structured artists, new fields).
- `src/app/(app)/create/page.test.tsx`, `src/app/(app)/create/confirm/page.test.tsx`: update mock track objects — remove `bpm`, `lastfmGenres`, `era`; add `spotifyPopularity`, `spotifyDurationMs`, `spotifyReleaseDate`, `derivedEra`, `lastfmTags`, `enrichmentVersion`, `enrichedAt`.

### PR 2: Chunked sync pipeline + artist repository

**Branch:** `feat/chunked-sync-pipeline`

Add chunked pagination to `fetchLikedSongs`, convert the sync pipeline to use chunked Inngest steps, and extract a standalone artist repository with `findStale` for Phase 2.

**Files (~6):**

1. `src/lib/spotify.ts` — chunked `fetchLikedSongs`
2. `src/repositories/artist.repository.ts` — **new** file
3. `src/inngest/functions/sync-library.ts` — chunked fetch + upsert steps
4. `tests/lib/spotify.test.ts` — chunking tests
5. `tests/inngest/functions/sync-library.test.ts` — chunked step tests
6. `tests/repositories/artist.repository.test.ts` — **new** test file

**Spotify chunking (`src/lib/spotify.ts`):**
- `fetchLikedSongs(token, opts?)`: add optional `{ startUrl?: string; maxTracks?: number }`. Returns `{ songs: SpotifyLikedSong[]; nextUrl: string | null }` instead of flat array. Stops after `maxTracks` songs or when pagination ends.

**Artist repository (`src/repositories/artist.repository.ts`):**
- `findStale(version: number, limit: number)` — finds artists where `enrichmentVersion < version`, used by Phase 2 enrichment steps

**Sync pipeline changes (`sync-library.ts`):**
- Step 3 (`fetch-songs`) → chunked loop: each invocation calls `fetchLikedSongs(token, { startUrl, maxTracks: 2000 })`, stores `nextUrl` as return value for next iteration. Step name includes offset: `fetch-songs-${offset}`.
- Step 4 (`upsert-songs` → rename to `upsert-data`) → chunked loop: process 500 songs per step. Step name includes offset: `upsert-data-${offset}`.

**Track repository:**
- Add `findStale(version: number, limit: number)` — finds tracks where `enrichmentVersion < version`, for Phase 2

**Test updates:**
- `fetchLikedSongs` tests: update for new `{ songs, nextUrl }` return shape, add tests for `startUrl` and `maxTracks` params
- Sync library tests: verify chunked step names, test loop continuation with `nextUrl`
- New artist repo tests: `findStale` query shape
